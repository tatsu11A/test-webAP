## 🧩フレームワークとは
Webアプリケーション開発に必要な仕組みをまとめて提供してくれるもの。
フレームワークには便利なライブラリが用意されているため、  
自分で一から作るよりも効率よく開発できる。
自動車の骨組み（フレーム）🚗  
※あとでイメージ図を追加予定

---

## ✅ メリット / ❌ デメリット

### メリット
- 便利な機能が最初から用意されている
- 開発効率が上がる
- コードの共通化ができ、チーム開発がしやすい

### デメリット
- 自由度が低い（ルールに従う必要がある）
---

## 🌱 Spring Frameworkについて
Spring Frameworkは、  
JavaでWebアプリケーション開発を行うためのフレームワーク。
ただし、機能ごとに分かれているため  
「どれを使えばいいか分かりづらい」という課題がある。

---

## 🚀 Spring Bootとは
上記の課題を解決するのが **Spring Boot**

### 特徴
- 設定を自動化してくれる
- 依存関係をまとめて管理してくれる
- 最小限のコードでWebアプリケーションを開発できる
---
## 🎯 今回の目的
Spring Bootを使用して、  
Webサーバー上で動作するタスク管理アプリケーションを開発する。
---

## 📌 実装する機能
- タスクの登録
- タスクの参照（一覧表示）
- タスクの編集
- タスクの削除

まずは最小限の機能で構築する。
---

## 🖥️ 前提環境
以下がインストール済みであること
- JDK 21
- JAVA_HOME の環境変数設定
---

## 🧱 プロジェクト作成

### 🔗 Spring Initializr
https://start.spring.io/

Spring Initializrは、無料でアプリケーションの雛形を作成できるサービス。

※あとでイメージ図を追加予定

---

### 📝 設定項目（概要）

| 項目 | 内容 |
|------|------|
| Project | ビルドシステム（Mavenなど） |
| Language | 使用言語 |
| Spring Boot | バージョン |
| Group | グループ名 |
| Artifact | プロジェクト名 |
| Package name | Javaのパッケージ名 |
| Packaging | jar / war |
| Java | Javaバージョン |
| Dependencies | 使用するライブラリ |

---

### 📦 プロジェクト生成手順

1. 「Generate」ボタンをクリック
2. zipファイルをダウンロード
3. 解凍する
src/main/java
<br>└ 起動クラス（mainメソッドを持つクラス）このクラスはSpring Initializrによって自動生成される。  
　　　　基本的には変更する必要はないです。
---
